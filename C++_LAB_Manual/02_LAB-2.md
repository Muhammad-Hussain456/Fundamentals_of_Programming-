### 2. Demonstrate Implicit and Explicit Type Conversions

**Problem Definition:** Show the difference between automatic (implicit) and manual (explicit) type conversions in C++.

**Problem Analysis:**
- Implicit conversion happens automatically by compiler
- Explicit conversion requires programmer intervention
- Potential data loss during conversions

**Algorithm:**
1. Start
2. Demonstrate implicit conversion
3. Demonstrate explicit conversion
4. Display results of all conversions
5. End

**Program:**
```cpp
#include <iostream>
using namespace std;

int main() {
    // Implicit Conversion
    int intVar = 5;
    float floatVar = intVar;
    cout << "Implicit conversion (int to float): " << intVar << " -> " << floatVar << endl;
    
    // Explicit Conversion
    float num = 3.7;
    int result1 = (int)num;
    float result2 = (float)intVar;
    
    cout << "Explicit conversion (float to int): " << num << " -> " << result1 << endl;
    cout << "Explicit conversion (int to float): " << intVar << " -> " << result2 << endl;

    return 0;
}
```

### 3. Program to Compute Simple Interest

**Problem Definition:** Calculate simple interest using the formula SI = (P × R × T) / 100.

**Problem Analysis:**
- Requires three inputs: principal, rate, time
- Simple arithmetic calculation
- Floating-point data type for accuracy

**Algorithm:**
1. Start
2. Read principal, rate, time
3. Calculate simple interest
4. Display result
5. End

**Program:**
```cpp
#include <iostream>
using namespace std;

int main() {
    float principal, rate, time, interest;
    
    cout << "Enter principal amount: ";
    cin >> principal;
    cout << "Enter rate of interest: ";
    cin >> rate;
    cout << "Enter time in years: ";
    cin >> time;

    interest = (principal * rate * time) / 100;
    cout << "Simple Interest: " << interest << endl;

    return 0;
}
```

### 4. Calculate Compound Interest

**Problem Definition:** Compute compound interest using the formula A = P(1 + R/100)^T.

**Problem Analysis:**
- Requires mathematical power function
- More complex than simple interest
- Need to calculate amount first, then interest

**Algorithm:**
1. Start
2. Read principal, rate, time
3. Calculate amount
4. Calculate compound interest
5. Display results
6. End

**Program:**
```cpp
#include <iostream>
#include <cmath>
using namespace std;

int main() {
    float principal, rate, time, amount, interest;
    
    cout << "Enter principal amount: ";
    cin >> principal;
    cout << "Enter rate of interest: ";
    cin >> rate;
    cout << "Enter time in years: ";
    cin >> time;

    amount = principal * pow((1 + rate / 100), time);
    interest = amount - principal;

    cout << "Compound Interest: " << interest << endl;
    cout << "Total Amount: " << amount << endl;

    return 0;
}
```
